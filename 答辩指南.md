# AI职业决策支持系统 - 答辩指南

---

## 一、项目一句话定义

**基于多智能体协作的AI职业规划平台**，通过5个专业Agent（职业规划、技能分析、副业推荐、简历优化、面试教练）为求职者提供个性化服务。

## 三、核心模块详解

### 3.1 智能体模块（agents/）

**作用**：实现AI智能体的核心逻辑

| 文件 | 作用 | 关键类/函数 |
|------|------|-------------|
| `base_agent.py` | Agent基类 | `BaseAgent`：ReAct推理、记忆集成、重试机制 |
| `career_agent.py` | 5个专业Agent | `CareerAgent`、`SkillAgent`、`SideJobAgent`、`ResumeAgent`、`InterviewAgent` |
| `orchestrator.py` | 编排器 | `AgentOrchestrator`：意图识别、任务拆分、DAG执行、结果合并 |
| `custom_llm.py` | 自定义LLM | `MiMoChatOpenAI`：支持mimo模型的reasoning_content |

**调用关系**：
```
用户输入
    ↓
Orchestrator（编排器）
    ↓ 意图识别
    ↓ 任务拆分
    ↓
Agent（智能体）
    ↓ ReAct推理
    ↓
Tool（工具）
    ↓ 查询数据库
    ↓
返回结果
```

---

### 3.2 工具模块（tools/）

**作用**：提供具体的功能实现，Agent通过调用工具完成任务

| 文件 | 工具数量 | 功能 |
|------|----------|------|
| `job_tools.py` | 5个 | 搜索职位、查询薪资、对比职位、保存目标岗位、更新档案 |
| `skill_tools.py` | 3个 | 技能差距分析、学习路径推荐、技能优先级排序 |
| `market_tools.py` | 3个 | 搜索副业、计算ROI、匹配度评分 |
| `resume_tools.py` | 6个 | 解析简历、分析JD、优化简历、ATS评分、生成简历、STAR改写 |
| `interview_tools.py` | 4个 | 生成面试题、优化自我介绍、薪资谈判建议、分析JD |

**工具调用示例**：
```python
@tool("search_jobs", return_direct=False)
def search_jobs(keyword: str, city: str = "") -> str:
    """搜索职位。输入职位名称和可选的城市。"""
    # 查询MySQL数据库
    jobs = Job.query.filter(Job.title.contains(keyword))
    if city:
        jobs = jobs.filter(Job.city == city)
    return jobs.limit(5).all()
```

---

### 3.3 记忆模块（memory/）

**作用**：让Agent记住用户信息和历史对话

| 文件 | 记忆类型 | 存储方式 | 生命周期 |
|------|----------|----------|----------|
| `short_term.py` | 短期记忆 | Python字典 | 单次对话 |
| `long_term.py` | 长期记忆 | ChromaDB向量数据库 | 永久 |

**三层记忆架构**：
```
L1 工作记忆（短期）
    - 当前对话上下文
    - 最近10轮消息
    - 存储在内存中

L2 情景记忆（长期）
    - 历史对话记录
    - 向量检索相关记忆
    - 存储在ChromaDB中

L3 语义记忆（用户档案）
    - 用户技能、目标岗位、偏好
    - 结构化存储
    - 存储在MySQL中
```

---

### 3.5 服务层（services/）

**作用**：处理业务逻辑，连接路由层和Agent层

**chat_service.py 核心功能**：
1. **SSE流式响应**：使用Server-Sent Events实现实时输出
2. **任务队列**：使用queue.Queue在后台线程和SSE生成器之间传递数据
3. **会话持久化**：保存对话历史到数据库

**SSE流程**：
```
前端请求 → Flask API → 创建任务队列 → 启动后台线程
                                    ↓
后台线程执行Agent → 通过queue传递步骤和token → SSE生成器发送到前端
                                    ↓
                              前端实时更新UI
```

## 四、核心流程图

### 4.1 单Agent处理流程

```
用户："搜索数据分析师岗位"
    ↓
Orchestrator.classify_intent()
    → 识别为 career 意图（置信度90%）
    ↓
Orchestrator._execute_single_agent("career", task)
    ↓
CareerAgent.run(task)
    → ReAct推理：Thought → Action → Observation
    → 调用 search_jobs 工具
    → 返回职位列表
    ↓
QualityAssessor.assess(output)
    → 质量分数：75分（合格）
    ↓
返回结果给前端
```

### 4.2 复合Agent处理流程

```
用户："搜索数据分析师岗位并出几道面试题"
    ↓
Orchestrator.classify_intent()
    → 识别为复合意图：career + interview（置信度90%）
    ↓
Orchestrator._split_composite_task()
    → 子任务1：career - 搜索数据分析师岗位
    → 子任务2：interview - 生成面试题
    ↓
Orchestrator._execute_subtasks()
    → 并行执行两个Agent
    ↓
ResultMerger.merge_with_llm()
    → LLM整合两个Agent的输出
    ↓
返回整合后的结果
```

## 六、答辩常见问题

| 问题 | 回答要点 |
|------|----------|
| "为什么用多Agent而不是单Agent？" | 领域专业化 + 提示词隔离 + 工具绑定 + 可独立优化 |
| "意图识别怎么做的？" | 两阶段：关键词匹配（快速） + LLM回退（精确） |
| "Agent之间怎么协作？" | DAG依赖图 + SharedContext共享上下文 |
| "记忆系统怎么设计的？" | 三层：工作记忆（短期） + 情景记忆（向量） + 语义记忆（档案） |
| "流式是怎么实现的？" | 后台线程 + Queue + SSE生成器，讲线程安全和打断机制 |
| "遇到过什么难点？" | mimo模型reasoning_content传回、PendingRollbackError、DOM多轮冲突 |
