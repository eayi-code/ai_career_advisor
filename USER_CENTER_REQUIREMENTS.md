# 用户中心需求文档

## 一、产品定位

**产品名称**：CareerAI - AI职业决策顾问
**用户中心作用**：让用户管理个人信息、查看使用记录、完善职业档案，以便AI提供更精准的职业建议

## 二、核心用户场景

### 场景1：新用户首次使用
```
注册 → 进入用户中心 → 完善基本信息 → 开始对话
```

### 场景2：老用户查看历史
```
登录 → 进入用户中心 → 查看对话历史 → 导出/继续对话
```

### 场景3：求职前完善档案
```
进入用户中心 → 完善职业档案（技能、目标岗位、求职意向）→ 开始对话获取精准建议
```

## 三、功能模块清单

### 模块1：用户信息头部（必须展示）
| 元素 | 说明 |
|------|------|
| 头像 | 点击可上传/更换 |
| 用户名 | 显示当前用户名 |
| 邮箱 | 显示注册邮箱 |
| 统计数字 | 总对话数、消息数、技能数量、工作经验 |

### 模块2：使用统计
- Agent使用分布（柱状图/饼图）
- 最近活跃时间线

### 模块3：对话历史
| 功能 | 说明 |
|------|------|
| 列表展示 | 显示对话标题、时间、消息数、使用的Agent |
| 继续对话 | 点击跳转到对应对话 |
| 导出对话 | 导出为文件 |
| 删除对话 | 删除单条记录 |

### 模块4：基本信息（账号管理）
| 字段 | 类型 | 说明 |
|------|------|------|
| 用户名 | 文本 | 可修改 |
| 邮箱 | 文本 | 只读 |
| 注册时间 | 日期 | 只读 |
| 密码 | 密码 | 可修改 |

### 模块5：职业档案（核心模块）

#### 5.1 基本信息
| 字段 | 类型 | 说明 |
|------|------|------|
| 学历 | 下拉选择 | 高中/大专/本科/硕士/博士 |
| 专业 | 文本 | 如：计算机科学 |
| 工作年限 | 数字 | 0-50年 |
| 当前职位 | 文本 | 如：软件工程师 |
| 技能 | 标签 | 逗号分隔，如：Python, JavaScript |

#### 5.2 求职意向
| 字段 | 类型 | 说明 |
|------|------|------|
| 目标岗位 | 文本 | 如：数据分析师 |
| 目标行业 | 文本 | 如：互联网, 金融 |
| 目标薪资范围 | 数字-数字 | 单位K，如：20-35 |
| 偏好城市 | 文本 | 如：北京, 上海 |
| 求职状态 | 下拉 | 观望中/在职看机会/已离职/应届生 |
| 工作偏好 | 下拉 | 不限/远程/坐班/混合 |
| 期望入职时间 | 下拉 | 随时/1个月内/3个月内/可商量 |
| 公司类型偏好 | 下拉 | 不限/大厂/创业公司/外企 |
| 历史目标岗位 | 标签列表 | 自动记录，显示添加时间 |

#### 5.3 项目经历（可动态添加）
| 字段 | 类型 | 说明 |
|------|------|------|
| 项目名称 | 文本 | 如：电商平台重构 |
| 角色 | 文本 | 如：前端负责人 |
| 技术栈 | 文本 | 如：React, Node.js |
| 成就 | 文本 | 如：性能提升50% |

#### 5.4 证书资质
| 字段 | 类型 | 说明 |
|------|------|------|
| 证书列表 | 标签 | 逗号分隔，如：PMP, AWS认证 |

#### 5.5 副业信息
| 字段 | 类型 | 说明 |
|------|------|------|
| 每周可用时间 | 数字 | 单位：小时 |
| 副业收入目标 | 数字 | 单位：元/月 |

#### 5.6 职业目标
| 字段 | 类型 | 说明 |
|------|------|------|
| 职业目标 | 长文本 | 自由填写长期职业规划 |

## 四、数据流向

```
用户填写档案 → 保存到数据库 → AI对话时读取档案 → 提供个性化建议
                                                      ↓
                              自动提取对话中的信息 → 更新档案（自动）
```

## 五、设计要求

### 视觉风格
- 参考Apple设计语言
- 分层折叠面板，平滑动画
- 白色背景 + 蓝色主题色 (#0066cc)
- SVG图标，不用emoji

### 交互要求
- 表单实时保存（或底部统一保存按钮）
- 折叠面板默认全部展开
- 项目经历可动态添加/删除
- 技能输入支持逗号分隔自动转换为标签

### 响应式
- 支持桌面端（主要）
- 可选支持平板端

## 七、给设计师的提示

1. **职业档案是核心** - 用户填写越完整，AI建议越精准
2. **减少填写负担** - 很多信息可从对话中自动提取，只展示不强制填写
3. **视觉层次清晰** - 用卡片/分组区分不同模块
4. **操作反馈明确** - 保存成功要有toast提示
5. **考虑空状态** - 新用户没有数据时的展示
